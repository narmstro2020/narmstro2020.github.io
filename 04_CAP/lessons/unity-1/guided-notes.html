üß± GameObjects & Components

In Unity, everything in your scene is a GameObject ‚Äî the player, the ground, the camera, lights, obstacles. But a GameObject by itself is just an empty container. Components are what give it behavior and appearance.

Unity uses composition over inheritance. Instead of one massive Player class with everything built in, you snap small, focused Components onto a GameObject ‚Äî like LEGO bricks. Need physics? Add a Rigidbody. Need custom behavior? Add your .cs script.

Common Built-in Components

ComponentWhat It Does
TransformPosition, rotation, and scale ‚Äî every GameObject has one
MeshFilterDefines the 3D shape (cube, sphere, custom mesh)
MeshRendererDraws the shape on screen with a material/color
ColliderInvisible hitbox for physics collisions
RigidbodyEnables physics (gravity, forces, momentum)
AudioSourcePlays sound effects or music
CameraRenders the scene from its viewpoint

Your Scripts Are Components Too!

When you write a C# script that inherits from MonoBehaviour, it becomes a custom component. You can drag it onto any GameObject ‚Äî just like you'd add a Rigidbody or Collider. Your public fields even appear as editable properties in the Inspector.

You already know how to write classes with properties and methods. A Unity component is exactly that ‚Äî a class. The only difference is it inherits from MonoBehaviour and Unity calls certain methods on it automatically.

SECTION 03
‚è± ~25 MIN

üîÑ MonoBehaviour ‚Äî Your Script's Lifecycle

Every Unity script inherits from MonoBehaviour. This base class gives your script access to Unity's lifecycle methods ‚Äî special methods that Unity calls automatically at specific moments during gameplay.

The Two Most Important Methods

using UnityEngine;

public class PlayerMove : MonoBehaviour
{
    public float speed = 5f;  // Visible in Inspector!

    // Called ONCE when the scene loads
    void Start()
    {
        Debug.Log("Game started! Player ready.");
    }

    // Called EVERY FRAME (~60 times per second)
    void Update()
    {
        // Game logic goes here ‚Äî movement, input, etc.
    }
}

Think of Start() like a constructor ‚Äî it runs once to set things up. Think of Update() like a while loop that runs forever (every frame) until the game stops. You've written loops that repeat; Update() is Unity's built-in game loop.

Other Lifecycle Methods (Preview)

You'll use these in future lessons, but it's good to know they exist:

MethodWhen It's Called
Awake()Very first thing ‚Äî even before Start()
Start()Once, right before the first Update()
Update()Every frame (variable timing)
FixedUpdate()At fixed intervals ‚Äî used for physics
OnCollisionEnter()When this object collides with another
OnDestroy()When this object is removed from the scene

public fields show up in the Inspector panel. This lets you tweak values (like speed) without changing code! But remember: the Inspector value overrides whatever you set in code. If you change speed = 5f to speed = 10f in code but the Inspector still says 5, Unity uses the Inspector value.

üìê Vector3, Transform & 3D Space

In 3D space, every position, direction, and size is described by three numbers: X, Y, and Z. Unity uses a struct called Vector3 to store these values.

Unity's Coordinate System

AxisDirectionColor in EditorVector3 Shorthand
XLeft / RightRedVector3.right ‚Üí (1, 0, 0)
YDown / UpGreenVector3.up ‚Üí (0, 1, 0)
ZBackward / ForwardBlueVector3.forward ‚Üí (0, 0, 1)

Creating a Vector3

// Create a specific position
Vector3 spawnPoint = new Vector3(0, 5, 0);  // 5 units up

// Handy shortcuts
Vector3 origin    = Vector3.zero;     // (0, 0, 0)
Vector3 up        = Vector3.up;       // (0, 1, 0)
Vector3 right     = Vector3.right;    // (1, 0, 0)
Vector3 forward   = Vector3.forward;  // (0, 0, 1)

The Transform Component

Every GameObject has a Transform. It stores the object's position, rotation, and scale ‚Äî all as Vector3 values. You access it in code with transform (lowercase). 

// Read the current position
Vector3 pos = transform.position;
Debug.Log($"Player is at: {pos}");

// Teleport to a new position
transform.position = new Vector3(0, 10, 0);

// Move smoothly every frame
transform.Translate(Vector3.right * speed * Time.deltaTime);

Why Time.deltaTime?

Update() runs every frame, but frames aren't evenly spaced. A fast computer might run 120 fps, a slow one 30 fps. Time.deltaTime is the time since the last frame ‚Äî multiplying by it ensures your object moves the same speed per second regardless of frame rate.

Without Time.deltaTime: movement depends on fps (faster computer = faster player).
With Time.deltaTime: movement is units per second (consistent everywhere).
