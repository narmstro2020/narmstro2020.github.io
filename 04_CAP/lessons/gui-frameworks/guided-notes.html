Why GUI Frameworks?

Everything you've built so far uses Console.ReadLine() and Console.WriteLine() ‚Äî sequential, blocking, text-only. A GUI framework gives you windows, buttons, sliders, and text fields driven by events.

WinForms: The Classic

Windows Forms (2002) ‚Äî the original C# GUI framework. Everything is pure C# ‚Äî there's no markup language. A drag-and-drop designer generates code to position controls with pixel coordinates.

üìñ Study This Code

Section02_WinFormsClassic.cs 

// WinForms: ALL layout in C#, pixel-positioned
var btn = new Button();
btn.Text = "Click Me";
btn.Location = new Point(50, 50);   // fixed pixels!
btn.Size = new Size(120, 40);

// Same += event pattern
btn.Click += (s, e) =>
    MessageBox.Show("Hello!");

this.Controls.Add(btn);

WinUI 3: Modern Windows

WinUI 3 is Microsoft's current recommended framework for Windows apps. It uses the same XAML concepts as WPF but with the Fluent Design System ‚Äî rounded corners, acrylic materials, and built-in dark mode.

üìñ Study This Code

Section04_WinUI3Modern.cs 

<!-- WinUI 3 XAML ‚Äî notice how similar to WPF! -->
<StackPanel Spacing="12" Padding="24">
  <TextBox x:Name="billBox" PlaceholderText="Enter bill" Header="Bill Amount"/>
  <Slider x:Name="tipSlider" Minimum="0" Maximum="30" Value="18"
          ValueChanged="OnTipChanged"/>
  <Button Content="Calculate" Click="OnCalculate"
          Style="{StaticResource AccentButtonStyle}"/>
</StackPanel>


// Code-behind ‚Äî same pattern as WPF
private void OnTipChanged(object sender, RangeBaseValueChangedEventArgs e)
{
    if (tipOutput != null)
        tipOutput.Text = $"Tip: {e.NewValue:F0}%";
}

private void OnCalculate(object sender, RoutedEventArgs e)
{
    if (decimal.TryParse(billBox.Text, out decimal bill))
    {
        decimal tip = bill * (decimal)(tipSlider.Value / 100);
        tipOutput.Text = $"Tip: {tip:C} | Total: {bill + tip:C}";
    }
}

XAML Fundamentals

XAML is XML-based: each tag creates a C# object, each attribute sets a property. Layout is controlled by panels ‚Äî containers that arrange child controls.

üìñ Study This Code

Section05_XamlFundamentals.cs

StackPanel ‚Äî stacks children vertically or horizontally:


<StackPanel Orientation="Vertical" Spacing="8">
  <TextBlock Text="First"/>
  <TextBlock Text="Second"/>
  <TextBlock Text="Third"/>
</StackPanel>

Grid ‚Äî rows and columns:


<Grid RowSpacing="8" ColumnSpacing="8">
  <Grid.RowDefinitions>
    <RowDefinition Height="Auto"/>     <!-- sizes to content -->
    <RowDefinition Height="*"/>        <!-- fills remaining -->
  </Grid.RowDefinitions>
  <Grid.ColumnDefinitions>
    <ColumnDefinition Width="150"/>    <!-- fixed 150px -->
    <ColumnDefinition Width="*"/>      <!-- fills rest -->
  </Grid.ColumnDefinitions>

  <TextBlock Text="Name:" Grid.Row="0" Grid.Column="0"/>
  <TextBox Grid.Row="0" Grid.Column="1"/>
</Grid>
üìè Sizing Keywords: Auto = size to content  |  * = fill remaining  |  2* = twice as much as 1*  |  150 = fixed pixels

Events in the GUI World

GUI events work exactly like the events from Unit 5: publisher.Event += handler;. The publisher is a control, and your handler is a method or lambda.

üìñ Study This Code

Section06_EventsInGuiWorld.cs

Way 1 ‚Äî Wire in XAML:


<Button Content="Go" Click="OnGoClicked"/>

// Must define a method with the matching signature:
private void OnGoClicked(object sender, RoutedEventArgs e) { }

Way 2 ‚Äî Wire in C# with +=:


// Lambda ‚Äî great for simple handlers
goBtn.Click += (s, e) => output.Text = "Clicked!";

// Named method ‚Äî better for complex logic
goBtn.Click += HandleGoClick;

Shared handler via sender:


// Multiple buttons, ONE handler
private void OnAnyClick(object sender, RoutedEventArgs e)
{
    var btn = (Button)sender;
    output.Text = $"Clicked: {btn.Content}";
}

Patterns & Architecture

GUI apps need structure. The partial class pattern connects XAML to code-behind, and model classes keep business logic separate from UI code.

üìñ Study This Code

Section07_PatternsArchitecture.cs

Separate model class (no UI knowledge):


public class TipCalculator
{
    public decimal BillAmount { get; set; }
    public double TipPercentage { get; set; }

    public decimal TipAmount
        => BillAmount * (decimal)(TipPercentage / 100);

    public decimal Total
        => BillAmount + TipAmount;
}

Thin code-behind ‚Äî Read ‚Üí Calculate ‚Üí Display:
