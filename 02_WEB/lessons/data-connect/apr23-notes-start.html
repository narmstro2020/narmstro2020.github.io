1Connecting JS to Supabase
How it works

Supabase provides a JavaScript library called supabase-js. We load it via a CDN script tag (no install needed), then create a client using our project URL and key. The client gives us methods like .select(), .insert(), .update(), and .delete() — these replace the SQL you've been writing manually.

// In your HTML <head>:
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>

// In your <script>:
const supabase = supabase.createClient("YOUR_URL", "YOUR_KEY");

TODO 4Initialize the Supabase client

Inside your <script> tag, create the client with your URL and anon key from TODO 2:

const SUPABASE_URL = "https://YOUR-PROJECT.supabase.co";
const SUPABASE_KEY = "eyJ...YOUR_ANON_KEY";
const db = supabase.createClient(SUPABASE_URL, SUPABASE_KEY);
We're naming it db to keep things short. This object is your connection to the database.
2Reading Data — .select()
JS equivalent of SELECT

Instead of writing SELECT * FROM guestbook, you write:

const { data, error } = await db
  .from('guestbook')
  .select('*')
  .order('created_at', { ascending: false });

data is an array of objects — just like the arrays you've been rendering to the DOM all semester. error is null if everything worked.

3Creating Data — .insert()
JS equivalent of INSERT

Instead of INSERT INTO guestbook (name, message) VALUES (...):

const { data, error } = await db
  .from('guestbook')
  .insert([{ name: "Jordan", message: "Hello!" }]);

Notice: .insert() takes an array of objects. Each object's keys must match your column names.

🗂️ Supabase JS Cheat Sheet
// READ (SELECT * FROM table)
const { data, error } = await db.from('table').select('*');

// CREATE (INSERT INTO table)
const { error } = await db.from('table').insert([{ col: 'val' }]);

// UPDATE (UPDATE table SET col = val WHERE id = X)
const { error } = await db.from('table').update({ col: 'val' }).eq('id', x);

// DELETE (DELETE FROM table WHERE id = X)
const { error } = await db.from('table').delete().eq('id', x);
