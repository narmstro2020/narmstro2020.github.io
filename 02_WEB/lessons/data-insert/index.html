1UPDATE — Changing Data
What it does

UPDATE changes values in rows that already exist. The basic shape:

UPDATE movies
SET rating = 9.5
WHERE title = 'The Matrix';
⚠️
The Golden Rule: Never run UPDATE or DELETE without a WHERE clause. Without one, you change EVERY row in the table. When in doubt, run a SELECT with your WHERE first to preview which rows it will affect.

2DELETE — Removing Rows
What it does

DELETE removes entire rows. You don't pick columns — DELETE removes the whole row that matches.

DELETE FROM movies
WHERE title = 'La La Land';

3JOINs — Combining Tables
Why JOIN?

Real databases split data across multiple tables to avoid repeating information. A movies table might store a director_id instead of repeating the director's name in every row. JOIN is how we glue the two tables back together when querying.

INNER JOIN Syntax
SELECT movies.title, directors.name
FROM movies
JOIN directors
  ON movies.director_id = directors.id;

Read it as: "Get title from movies and name from directors, where the movie's director_id matches the director's id."
