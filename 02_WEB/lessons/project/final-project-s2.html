Project Overview

Take your existing multi-page website from Semester 1 and bring it to life with JavaScript interactivity and a Supabase database backend. Your S1 HTML/CSS stays — you're adding new functionality on top.

By the end, your site will:

  • Read data from a Supabase database and render it to the page
  • Let users submit data through a form that saves to Supabase
  • Use JavaScript for DOM manipulation, event handling, and dynamic content
  • Maintain all your existing Semester 1 CSS/responsive design
Tools

WebStorm · GitHub · Supabase · Browser Dev Tools

1Technical Requirements

A. JavaScript Fundamentals

Your project must demonstrate these JS concepts somewhere in your site:

  • Variables & Data Types — const, let, strings, numbers, booleans
  • Conditionals — at least one if/else or switch controlling behavior
  • Arrays and/or Objects — storing and accessing structured data
  • Loops — forEach, for, or for...of to iterate and render data
  • Functions — at least 3 named functions that organize your logic
  • DOM Manipulation — querySelector, getElementById, innerHTML or textContent
  • Event Listeners — at least 2 different events (e.g., click, submit, input)

B. Supabase Database Integration

Your site must connect to a Supabase database and perform at minimum:

  • READ (.select) — Load data from at least one Supabase table and render it to the DOM
  • CREATE (.insert) — A form that submits new data to your Supabase table
  • Async/Await — All Supabase calls must use async/await
  • Error Handling — Check for errors on every Supabase call
  • Loading State — Show a loading indicator while data is being fetched
Stretch goals (extra credit): Add UPDATE (.update) and/or DELETE (.delete) functionality — edit or remove entries from the page + database.

C. Database Design

  • At least one Supabase table with 3+ columns (not counting id and created_at)
  • Table must have a primary key (auto-increment id)
  • Table must have a timestamp column (created_at DEFAULT NOW())
  • Row Level Security disabled for class practice
Stretch goal: Use two related tables with a foreign key, and display joined data.

D. Code Quality & Organization

  • JavaScript in external .js file(s) — no inline onclick attributes (event listeners only)
  • Use comments to explain major sections of your JS
  • Meaningful variable and function names (no x, thing, data2)
  • Supabase URL and key stored in clearly labeled constants at the top of the file
  • console.log statements removed before submission (or commented out)

E. Git & GitHub

  • Work on a final-project-s2 branch (or clearly named equivalent)
  • At least 7 meaningful commits — one per work day minimum
  • Clear commit messages describing what changed (not "update" or "stuff")
  • Final push must leave the repository in a working state

Supabase JS Cheat Sheet
// Setup (in HTML <head>)
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>

// Initialize (top of your JS file)
const SUPABASE_URL = "https://YOUR-PROJECT.supabase.co";
const SUPABASE_KEY = "YOUR_ANON_KEY";
const db = supabase.createClient(SUPABASE_URL, SUPABASE_KEY);

// READ
const { data, error } = await db.from('table').select('*');

// CREATE
const { error } = await db.from('table').insert([{ col: 'val' }]);

// UPDATE
const { error } = await db.from('table').update({ col: 'val' }).eq('id', x);

// DELETE
const { error } = await db.from('table').delete().eq('id', x);
