Part A · Async / Await
PART A — CLEANER SYNTAX
What is async/await?

async/await is syntactic sugar over Promises. Instead of chaining .then(), we use the await keyword to pause until a Promise resolves. The function must be marked async.

// .then() chain (what you know)
fetch("url").then(r => r.json()).then(d => console.log(d));

// async/await (cleaner version)
async function getData() {
  const response = await fetch("url");
  const data = await response.json();
  console.log(data);
}

Part A · Try / Catch Error Handling
try / catch replaces .catch()

With async/await, we wrap our code in a try { } block. If anything fails, execution jumps to the catch { } block — just like .catch() on a Promise chain.

async function getData() {
  try {
    const res = await fetch("url");
    const data = await res.json();
    // use data...
  } catch (error) {
    console.error("Failed:", error);
  }
}

Part B · POST Requests
PART B — SENDING DATA
GET vs POST

GET = "give me data" (what we've been doing).
POST = "here's some data, save it." POST requests include a body — the data you're sending — and a Content-Type header telling the server the format.

const response = await fetch("url", {
  method: "POST",
  headers: {
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    title: "My Post",
    body: "Hello world",
    userId: 1
  })
});
Key Pieces of a POST request

method: "POST" — tells the server we're sending, not requesting.
headers: "Content-Type": "application/json" — tells the server the body is JSON.
body: JSON.stringify(object) — converts our JS object to a JSON string for transport.
