Section 1 · What Is an API?
API — Application Programming Interface

An API is a way for two programs to talk to each other. When your JavaScript asks a server for data, it's using an API.

Think of it like a restaurant: you (your code) place an order using the menu (the API docs), and the kitchen (the server) sends back food (data). You don't need to know how the kitchen works — just what to order.

JSON — JavaScript Object Notation

APIs return data in JSON format. It looks almost identical to a JavaScript object — keys in double quotes, values as strings, numbers, arrays, or nested objects.

// A JSON response from an API looks like this:
{
  "id": 1,
  "name": "Leanne Graham",
  "email": "sincere@april.biz",
  "company": {
    "name": "Romaguera-Crona"
  }
}

Section 2 · The fetch() Function
fetch(url) — sending a request

fetch(url) sends an HTTP request to a server and immediately returns a Promise — a placeholder for data that will arrive in the future. Your code doesn't stop and wait; it keeps running. When the data arrives, your .then() callback runs.

Section 3 · Promises & .then()
What is a Promise?

A Promise represents a value that isn't available yet. It has three states:

• pending — waiting for the server to respond

• fulfilled — response arrived, .then() runs

• rejected — something went wrong, .catch() runs

Why TWO .then() calls?

The first .then() receives the raw Response object — headers, status code, etc. — but not the data itself. Calling .json() on it reads and parses the body, which is itself async (another Promise).

The second .then() finally receives your parsed JavaScript data.

// Step-by-step: what each .then() receives
fetch("https://jsonplaceholder.typicode.com/users")

  .then(response => {
    console.log(response.status);  // 200
    console.log(response.ok);      // true
    return response.json();         // ← returns another Promise
  })

  .then(data => {
    console.log(data);               // ← your array/object ✓
  });

Section 4 · Rendering API Data to the DOM
Same rendering pattern — new data source

Once you have data inside the second .then(), use the exact same pattern from last unit: loop with forEach, build an HTML string, and set innerHTML.

fetch("https://jsonplaceholder.typicode.com/users")
  .then(res => res.json())
  .then(users => {
    const grid = document.querySelector("#user-grid");
    let html = "";

    users.forEach(user => {
      html += `
        <div class="user-card">
          <div class="uc-name">${user.name}</div>
          <div class="uc-email">${user.email}</div>
        </div>`;
    });

    grid.innerHTML = html;
  });

Section 5 · Inspecting the Raw Response
JSON.stringify()

JSON.stringify(obj, null, 2) converts a JavaScript object back into a formatted JSON string. The null and 2 arguments tell it to indent with 2 spaces — great for reading the structure of API responses.

// Convert JS object → JSON string
const str = JSON.stringify(data, null, 2);
console.log(str);

// Convert JSON string → JS object
const obj = JSON.parse(str);

Section 6 · Error Handling with .catch()
⚠️
Network requests can fail — bad URL, no internet, server down. Always add .catch() so your app doesn't silently break.
fetch("https://bad-url.invalid/data")
  .then(res => res.json())
  .then(data => { /* ... */ })
  .catch(err => {
    console.error("Fetch failed:", err.message);
    // update the DOM to show an error to the user
  });
