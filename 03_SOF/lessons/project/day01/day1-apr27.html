Today's Goal

By the end of class, you should have a running libGDX project with a player that moves, jumps, and lands on the ground — pushed to GitHub. This is the skeleton everything else builds on.

🚨 Do NOT skip the GitHub setup. If your project isn't on GitHub by the end of today, you risk losing all your work. Set it up first, code second.

1 Create Your Project (~15 min)

Option A — Start Fresh (recommended for a clean project)

  1. Go to gdx-liftoff or use the gdx-liftoff tool
  2. Settings:
    • Project name: Your game name (e.g. PlatformerGame)
    • Package: io.github.yourusername
    • Platforms: Check lwjgl3 (desktop) only
    • Java version: 17 or higher
  3. Generate and open in IntelliJ
  4. Wait for Gradle to finish importing (bottom progress bar)
  5. Run the lwjgl3 launcher to verify it works (you should see the libGDX logo)

Option B — Continue from Activities

You can build on top of your existing project from the animation and gameplay activities. Just rename/reorganize:

  1. Keep your existing project
  2. Delete or rename AnimationDemoScreen.java, AnimationActivityScreen.java, and GameplayScreen.java — you'll build fresh versions
  3. Keep the sprite PNGs in assets/

⚠️ Make sure your build.gradle has Java 17: Open core/build.gradle and verify sourceCompatibility = 17 and targetCompatibility = 17. If they say 1.8, change them now.

3 Set Up Main.java (~5 min)

Your Main.java should extend Game (not ApplicationAdapter) so you can use setScreen().

package io.github.yourusername;

import com.badlogic.gdx.Game;

public class Main extends Game {
    @Override
    public void create() {
        setScreen(new GameScreen(this));
    }
}

💡 Passing this to GameScreen gives it a reference to the Game object, which it will need later for screen transitions (e.g. game.setScreen(new GameOverScreen(game))).

4 Build GameScreen.java (~45 min)

This is where you'll spend most of today. Build it piece by piece — run after each piece to make sure it works.

4a — Skeleton

package io.github.yourusername;

import com.badlogic.gdx.*;
import com.badlogic.gdx.graphics.*;
import com.badlogic.gdx.graphics.g2d.*;
import com.badlogic.gdx.math.Rectangle;

public class GameScreen implements Screen {

    private final Game game;

    // Constants
    private static final float GRAVITY = -500f;
    private static final float JUMP_VELOCITY = 300f;
    private static final float MOVE_SPEED = 150f;
    private static final float GROUND_Y = 50f;

    // Rendering
    private SpriteBatch batch;
    private OrthographicCamera camera;
    private Texture playerTexture;

    // Player state
    private float playerX = 100f;
    private float playerY = GROUND_Y;
    private float velocityY = 0f;
    private boolean onGround = true;

    public GameScreen(Game game) {
        this.game = game;
    }

    @Override
    public void show() {
        // Step 4b goes here
    }

    @Override
    public void render(float delta) {
        // Steps 4c, 4d, 4e go here
    }

    @Override public void resize(int w, int h) { camera.setToOrtho(false, w, h); }
    @Override public void pause() {}
    @Override public void resume() {}
    @Override public void hide() {}

    @Override
    public void dispose() {
        batch.dispose();
        playerTexture.dispose();
    }
}

Run it. It should compile (but crash because show() is empty). That's fine — you'll fill it in next.

4b — Initialize in show()

@Override
public void show() {
    batch = new SpriteBatch();
    camera = new OrthographicCamera();
    camera.setToOrtho(false, 640, 480);

    // For today, use a simple placeholder texture.
    // You'll replace this with a sprite sheet on Day 2.
    playerTexture = new Texture("player.png");
}

Run it. You should see nothing on screen (haven't drawn yet). No crash = good.

4c — Draw the player in render()

@Override
public void render(float delta) {
    Gdx.gl.glClearColor(0.1f, 0.1f, 0.15f, 1f);
    Gdx.gl.glClear(GL20.GL_COLOR_BUFFER_BIT);

    batch.setProjectionMatrix(camera.combined);
    batch.begin();
    batch.draw(playerTexture, playerX, playerY, 64, 64);
    batch.end();
}

Run it. You should see your player sprite on screen, frozen in place.

4d — Add input handling

Add this before the drawing code in render():

// ── Input ──
if (Gdx.input.isKeyPressed(Input.Keys.LEFT)) {
    playerX -= MOVE_SPEED * delta;
}
if (Gdx.input.isKeyPressed(Input.Keys.RIGHT)) {
    playerX += MOVE_SPEED * delta;
}
if (Gdx.input.isKeyPressed(Input.Keys.SPACE) && onGround) {
    velocityY = JUMP_VELOCITY;
    onGround = false;
}

Run it. LEFT/RIGHT should move the player. SPACE won't do anything visible yet (gravity isn't applied).

4e — Add gravity and ground collision

Add this after the input code, before drawing:

// ── Physics ──
velocityY += GRAVITY * delta;
playerY += velocityY * delta;

if (playerY <= GROUND_Y) {
    playerY = GROUND_Y;
    velocityY = 0;
    onGround = true;
}

Run it. The player should now walk, jump, fall with gravity, and land on the ground. This is your foundation — everything builds on this.
