Requirements 100 PTS

Core Gameplay (40 pts)

  • Player with keyboard-controlled movement (LEFT/RIGHT + SPACE to jump)
  • Gravity pulls player down; ground collision prevents falling through
  • At least 2 patrolling enemies that move back and forth
  • Player-enemy collision has a consequence (lose life, reset, etc.)
  • At least 5 collectible items that disappear when collected
  • Score tracking that increases on collection
  • At least 3 platforms at different heights to jump between
  • A win condition or end-of-level trigger

Animation & Visuals (20 pts)

  • Player has at least 3 animation states (idle, run, jump) via sprite sheets
  • Animations switch correctly based on player state
  • Player sprite flips to face direction of movement
  • Enemies and/or collectibles are animated
  • Screen clears and redraws properly (no visual artifacts)

Screens & UI (20 pts)

  • MenuScreen — title, start instructions, transitions to gameplay
  • GameScreen — main gameplay
  • GameOverScreen — final score, restart or menu option
  • HUD on GameScreen showing at least score and lives
  • All screen transitions work without crashes

Code Quality (10 pts)

  • All textures, batches, and fonts properly dispose()d
  • Named constants instead of magic numbers (GRAVITY, MOVE_SPEED)
  • Code organized into logical methods, not everything in render()
  • Meaningful variable and method names
  • Compiles and runs without errors

Completeness (10 pts)

  • Game is playable from start to finish (menu → play → game over → restart)
  • All checklist items above are present and functional

Bonus (up to +10 pts)

  • +3 — Sound effects (jump, coin, death) and/or background music
  • +3 — Multiple levels or level progression
  • +2 — Save/load high score using File I/O (Gson or plain text)
  • +2 — Additional enemy types with different behaviors

HUD Quick Start — BitmapFont

libGDX includes a default font. Here's the minimal code to draw text on screen:

// Field
BitmapFont font;

// In show()
font = new BitmapFont();
font.setColor(Color.WHITE);

// In render(), inside batch.begin/end
font.draw(batch, "Score: " + score, 10, 470);
font.draw(batch, "Lives: " + lives, 10, 450);

// In dispose()
font.dispose();

💡 Important: HUD text should use screen coordinates, not world coordinates. If your camera moves with the player, either draw the HUD with a separate fixed camera, or draw it after resetting the projection matrix.
