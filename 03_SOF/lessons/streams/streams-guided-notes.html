What Is a Stream?

Definition

A Stream in Java is a sequence of elements that supports chained operations to process data from a source (like a list or array). Streams let you filter, transform, and reduce collections of data without writing manual loops.

⚠️
Not the Same as File I/O!

Java has two different things called "streams." InputStream / OutputStream deal with reading files and network data. java.util.stream.Stream is for processing collections. We're talking about the second kind today.

The Big Idea: Pipelines

A Stream works like a factory assembly line. Data flows in from a source, passes through one or more intermediate operations (each producing a new stream), and finally exits through a terminal operation that produces a result.

💡
Key Rule:

Each intermediate operation returns a new Stream — so you can chain them. A terminal operation consumes the stream and returns a final result (a list, a number, void, etc.). Once consumed, the stream is done — you can't reuse it.

2

Creating a Stream

Common Ways to Get a Stream

The most common source is a List or other Collection:

import java.util.List;
import java.util.stream.Stream;

List<String> names = List.of("Mario", "Luigi", "Peach", "Bowser");

// Option 1: .stream() on any Collection
names.stream()

// Option 2: Stream.of() for quick inline streams
Stream.of("Mario", "Luigi", "Peach")

// Option 3: Arrays.stream() for arrays
int[] scores = {100, 200, 50};
Arrays.stream(scores)
3

Intermediate Operations

What They Do

Intermediate operations transform or filter elements and always return a new Stream. They are lazy — they don't run until a terminal operation is called.

Method Type What It Does Example
filter() Intermediate Keeps elements that match a condition (Predicate) .filter(n -> n > 0)
map() Intermediate Transforms each element into something else (Function) .map(s -> s.toUpperCase())
sorted() Intermediate Sorts elements in natural or custom order .sorted()
distinct() Intermediate Removes duplicate elements .distinct()
limit(n) Intermediate Takes only the first n elements .limit(5)
peek() Intermediate Inspects each element without changing it (great for debugging) .peek(System.out::println)
🔗
Lambdas and Functional Interfaces

Each stream operation takes a functional interface as its argument — Predicate for filter(), Function for map(), Consumer for forEach(). That's why you pass lambda expressions like x -> x > 0 directly into them!

4

Terminal Operations

What They Do

Terminal operations trigger the pipeline and produce a final result — a value, a list, or a side effect like printing. After a terminal operation, the stream is consumed and cannot be used again.

Method Type Returns Use Case
forEach() Terminal void Run an action on each element
collect() Terminal Collection (List, Set, Map) Gather results into a new list
count() Terminal long Count matching elements
findFirst() Terminal Optional<T> Get the first element (may not exist)
anyMatch() Terminal boolean True if any element matches condition
reduce() Terminal Optional<T> Combine all elements into one value
📦
The Most Common Terminal: collect(Collectors.toList())

To turn a stream back into a usable list, use: .collect(Collectors.toList()). You'll also see Collectors.toUnmodifiableList() and Collectors.joining() for Strings.

5

Putting It Together — Full Examples

Example A — Filter & Collect

Get all names that start with "B" and collect them into a new list:

List<String> names = List.of("Bowser", "Mario", "Boo", "Peach", "Birdo");

List<String> bNames = names.stream()
    .filter(name -> name.startsWith("B"))
    .collect(Collectors.toList());

// bNames → ["Bowser", "Boo", "Birdo"]

Example B — Map & Collect

Transform a list of integers into their squared values:

List<Integer> nums = List.of(1, 2, 3, 4, 5);

List<Integer> squares = nums.stream()
    .map(n -> n * n)
    .collect(Collectors.toList());

// squares → [1, 4, 9, 16, 25]

Example C — Filter + Map + Count

How many enemies have more than 50 HP?

List<Enemy> enemies = /* ... */;

long toughCount = enemies.stream()
    .filter(e -> e.getHp() > 50)
    .count();

System.out.println("Tough enemies: " + toughCount);

Example D — Sorted + Limit + forEach

Print the top 3 highest scores:

List<Integer> scores = List.of(400, 100, 850, 300, 720);

scores.stream()
    .sorted((a, b) -> b - a)   // descending
    .limit(3)
    .forEach(System.out::println);

// prints: 850, 720, 400
6

Lazy Evaluation

Streams Don't Run Until Asked

Intermediate operations are lazy — they build up a plan but don't execute it until a terminal operation triggers the pipeline. This makes streams efficient: elements that don't need to be processed won't be.

// This builds a pipeline but runs NOTHING yet:
Stream<String> pipeline = names.stream()
    .filter(n -> {
        System.out.println("Filtering: " + n);
        return n.length() > 4;
    });

// Nothing prints yet! The pipeline hasn't run.

// NOW it runs (terminal operation triggers everything):
pipeline.forEach(System.out::println);
⚡
Why does this matter?

If you use limit(1) after a filter, Java only processes elements until one match is found — it doesn't process the entire list. This is called short-circuiting.

Streams in Your Platformer

🎮 Platformer Connection

Your game is full of collections — lists of enemies, coins, platforms, and projectiles. Streams let you query and transform those collections without messy loops. Here are some places you'll use them:

// 1. Find all active enemies still on screen
List<Enemy> activeEnemies = enemies.stream()
    .filter(Enemy::isAlive)
    .filter(e -> e.isOnScreen(camera))
    .collect(Collectors.toList());

// 2. Count remaining coins in the level
long coinsLeft = coins.stream()
    .filter(c -> !c.isCollected())
    .count();

// 3. Get names of all platforms sorted by x-position
List<String> sortedPlatforms = platforms.stream()
    .sorted((a, b) -> Float.compare(a.getX(), b.getX()))
    .map(Platform::getName)
    .collect(Collectors.toList());
