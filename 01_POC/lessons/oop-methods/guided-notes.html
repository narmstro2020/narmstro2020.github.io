What Are Methods?

11:09 – 11:19 (10 min)

👨‍🏫 Instructor Demo

In U3B1 we learned about attributes — data an object HAS (like name, hp, level). Now we add methods — actions an object CAN DO.

A method is simply a function defined inside a class. It always takes self as its first parameter so it can access the object's own data.

class Hero: def __init__(self, name, hp): self.name = name self.hp = hp self.level = 1 hero = Hero("Aria", 100) print(f"{hero.name} has {hero.hp} HP") # Aria has 100 HP — but she can't DO anything yet!
2

Defining Instance Methods

11:19 – 11:29 (10 min)

👨‍🏫 Instructor Demo

Define methods with def inside the class, indented one level. Always include self as the first parameter. When you call the method, Python passes self automatically.

class Hero: def __init__(self, name, hp): self.name = name self.hp = hp def attack(self): print(f"{self.name} swings their sword!") def war_cry(self): print(f"{self.name} shouts: 'For glory!'") hero = Hero("Aria", 100) hero.attack() # Aria swings their sword! hero.war_cry() # Aria shouts: 'For glory!'

Methods with Parameters & Return

11:29 – 11:39 (10 min)

👨‍🏫 Instructor Demo

Methods can take extra parameters (besides self), return values, and modify the object's state by updating self.attribute.

def take_damage(self, amount): # extra param self.hp -= amount # modifies state def is_alive(self): # return value return self.hp > 0 def level_up(self): # modifies state self.level += 1 self.hp += 20

Inheritance Basics

11:44 – 11:54 (10 min)

👨‍🏫 Instructor Demo

Inheritance lets a child class get everything from a parent class, then add its own stuff. Put the parent's name in parentheses: class Warrior(Character):

class Character: # parent def __init__(self, name, hp): self.name = name self.hp = hp def get_info(self): return f"{self.name} (HP: {self.hp})" class Warrior(Character): # child inherits! def __init__(self, name, hp, strength): self.name = name self.hp = hp self.strength = strength def slash(self): print(f"{self.name} slashes for {self.strength} damage!") w = Warrior("Thorin", 120, 85) print(w.get_info()) # inherited from Character!

The super() Function

11:54 – 12:04 (10 min)

👨‍🏫 Instructor Demo

In Section 4 we repeated self.name = name and self.hp = hp in the child. That's not DRY! super().__init__() calls the parent's __init__ so you don't repeat yourself.

class Warrior(Character): def __init__(self, name, hp, strength): super().__init__(name, hp) # parent handles name & hp self.strength = strength # child adds its own

Pygame Sprites & Groups

12:04 – 12:14 (10 min)

👨‍🏫 Instructor Demo

pygame.sprite.Sprite is a parent class provided by Pygame. Your game objects inherit from it — exactly like Section 4! It gives you self.image, self.rect, .kill(), and .update(). A Group manages multiple sprites at once.

class Player(pygame.sprite.Sprite): def __init__(self, x, y): super().__init__() # set up Sprite internals self.image = pygame.Surface((32, 32)) self.image.fill("green") self.rect = self.image.get_rect(center=(x, y)) # Group manages them: all_sprites = pygame.sprite.Group() all_sprites.add(player) all_sprites.update() # calls each sprite's update() all_sprites.draw(screen) # draws them all

The update() Pattern

12:14 – 12:24 (10 min)

👨‍🏫 Instructor Demo

In Pygame, every sprite can have its own update() method. The game loop calls group.update() which triggers each sprite's update() individually. Different sprites can have different behavior — same method name, different implementation!

# Player moves based on keyboard input class Player(GameSprite): def update(self): self.x += self.speed # moves right # Monster moves automatically class Monster(GameSprite): def update(self): self.x -= self.speed # moves left
