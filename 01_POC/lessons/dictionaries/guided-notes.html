What Are Dictionaries?

11:09 ‚Äì 11:19

A dictionary stores data as key‚Äìvalue pairs. Instead of accessing items by index (like a list), you access them by a unique key.

üéÆ Analogy: Think of a video game stats screen ‚Äî you don't say "give me item at index 0." You say "give me the health." The stat name is the key, the number is the value.

üíª Instructor Demo

section01_what_are_dictionaries.py 
# A dictionary stores data as key-value pairs
# Syntax: {key: value, key: value, ...}
player = {
    "name": "Aria",
    "health": 100,
    "attack": 25
}
print(player)
print(f"Player name: {player['name']}")

# Keys must be unique ‚Äî duplicates overwrite!
demo = {"a": 1, "b": 2, "a": 99}
print(demo)  # "a" is 99, not 1

Accessing & Modifying Values

11:19 ‚Äì 11:29

Use brackets [ ] when you're sure the key exists. Use .get() for safe access that won't crash.

‚ö†Ô∏è Warning: dict["missing_key"] crashes with KeyError! Use .get() to be safe.

üíª Instructor Demo

section02_accessing_and_modifying.py 
player = {"name": "Aria", "health": 100, "attack": 25}
print(player.get("mana"))          # None (no crash!)
print(player.get("mana", 0))       # 0 (custom default)
player["health"] = 80             # Update
player["mana"] = 50               # Add new key
del player["attack"]              # Delete

Common Dictionary Methods

11:29 ‚Äì 11:39

Python dicts have .keys(), .values(), .items(), the in operator, and .pop().

üí° Tip: The in operator checks keys by default, NOT values. Use value in dict.values() for values.

üíª Instructor Demo

section03_common_methods.py 
player = {"name": "Aria", "health": 100, "attack": 25, "defense": 15}
print(list(player.keys()))       # all keys
print(list(player.values()))     # all values
print(list(player.items()))      # all (key, value) pairs
print("health" in player)         # True
print(100 in player.values())     # True
print(len(player))                 # 4
removed = player.pop("defense")   # removes and returns 15

Looping Through Dictionaries

11:44 ‚Äì 11:54

Loop over keys (default), values, or both with .items(). The .items() loop is the most common.

üìå Quick Ref: Keys ‚Üí for k in dict ¬∑ Values ‚Üí for v in dict.values() ¬∑ Both ‚Üí for k, v in dict.items()

üíª Instructor Demo

section04_looping.py 
inventory = {"sword": 1, "potion": 5, "shield": 1, "arrow": 20}

for item in inventory:              # keys only
    print(item)

for count in inventory.values():   # values only
    print(count)

for item, count in inventory.items():  # both ‚≠ê
    print(f"{item}: {count}")

Nested Dictionaries

11:54 ‚Äì 12:04

A dictionary value can be another dictionary! Chain brackets: dict[outer_key][inner_key].

üíª Instructor Demo

section05_nested_dictionaries.py 
party = {
    "warrior": {"name": "Kael", "health": 150, "attack": 40},
    "mage":    {"name": "Lyra", "health": 80,  "attack": 60}
}
print(party["warrior"]["name"])     # "Kael"
party["mage"]["health"] = 60       # modify nested

for role, stats in party.items():
    print(f"--- {role} ---")
    for key, val in stats.items():
        print(f"  {key}: {val}")

Practical Patterns

12:14 ‚Äì 12:24

Three essential patterns: counting occurrences, lookup tables, and building dicts from user input.

üíª Instructor Demo

section07_practical_patterns.py 

# Pattern 1: Counting
loot_drops = ["gold", "gem", "gold", "potion", "gold", "gem"]
counts = {}
for item in loot_drops:
    if item in counts:
        counts[item] += 1
    else:
        counts[item] = 1
print(counts)  # {"gold": 3, "gem": 2, "potion": 1}

# Pattern 2: Lookup Table
element_chart = {"fire": "Strong vs Ice", "water": "Strong vs Fire"}
advantage = element_chart.get("fire", "Unknown")
print(advantage)  # "Strong vs Ice"

# Pattern 3: Building from input
character = {}
character["name"] = input("Name: ")
character["class"] = input("Class: ")
character["level"] = int(input("Level: "))
